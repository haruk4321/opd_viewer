# opd_viewer
# OPD システムビューアー

Web上の製品記述とユーザーレビューから、製品のシステムモデル図（OPD: Object-Process Diagram）を自動生成する設計支援システムの、**モデル可視化ビューアー**です。

**公開ページ:** https://haruk4321.github.io/opd_viewer/

## 背景

製品開発では、構造・機能・ユーザー要求を整理するモデルベース設計の重要性が高まっていますが、モデルの作成には専門知識と多大な労力が必要です。本研究では、Web上の製品記述とユーザーレビューを自然言語処理・LLMで構造化し、設計要素とユーザー評価を結び付けたモデル図を自動生成する手法を構築しました。本ビューアーは、その生成結果を確認・探索するためのフロントエンドです。

## システム構成

```
[Web上の製品記述・ユーザーレビュー]
            │  スクレイピング
            ▼
[ Python：自然言語処理 + LLM API ]
   ・機能／構造要素の抽出
   ・ユーザー要求の抽出、重要度・ポジティブ率の算出
            │
            ▼
[ CSV：OPM要素・関係・要求 ]
            │  OPD_generate_gui.py
            ▼
[ HTML + Cytoscape.js ビューアー ]  ← 本リポジトリ
```

## 使用技術

| 領域 | 技術 |
| --- | --- |
| データ生成 | Python, pandas, 自然言語処理（GiNZA / MeCab / WordNet）, LLM API |
| 可視化 | HTML, CSS, JavaScript, Cytoscape.js |

## ビューアーの機能

- **要素の種別を形状で表現** — Process（楕円）／Object（角丸四角）／外部要因（八角形）
- **ユーザー評価を枠線で表現** — 色：加重平均ポジティブ率（赤＝ポジティブ／青＝ネガティブ）、太さ：関連要求の合計重要度
- **OPMのリンク種別を描き分け** — Consumption / Result / Effect / Instrument / Agent / Enabling / Aggregation / Exhibition / Specialization / Instantiation を線種・矢尻・色で表現
- **クリックによる詳細表示** — ノードを選択すると、OPLの説明文、関連する要求、加重平均ポジティブ率、合計重要度を右パネルに動的表示
- **動的な凡例生成** — 実際にモデル内で使われている関係タイプのみを凡例に表示

## 設計上の工夫

- **設計要素とユーザー評価の対応付け** — どの部品・機能がユーザーにどう評価されているかを、図上の色と太さだけで把握できるようにした
- **情報量のコントロール** — 詳細情報を常時表示するとモデル図が読めなくなるため、クリック時にパネルへ出す構成とし、図の可読性を保った
- **凡例の自動生成** — 使われていない関係タイプを凡例から除くことで、専門知識がない人でも図を読み解けるようにした
